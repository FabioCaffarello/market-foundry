# StrategyResolvedEvent Producer Spec and Derive Ownership Model

> **Stage**: S365 — Producer Spec and Derive Ownership (DI-1)
> **Wave**: Derive Integration Wave (S364–S369)
> **Status**: Binding
> **Date**: 2026-03-22

---

## 1. Executive Summary

This document specifies how the derive binary produces `StrategyResolvedEvent`
for the `mean_reversion_entry` family. It maps derive's actual implementation
against the S359 canonical contract, defines ownership rules, and provides a
field-level compliance matrix covering all 11 binding invariants.

**Verdict**: The derive producer is **contract-compliant** with zero blocking
mismatches. One minor deviation in the deduplication key prefix (`strat:` vs
`exec:`) is intentional and correct — the `strat:` prefix belongs to the
producer domain, while `exec:` belongs to the consumer domain. Each domain
generates its own dedup key for its own stream.

## 2. Producer Model

### 2.1 Producer Chain

The derive binary produces `StrategyResolvedEvent` through a deterministic
actor chain within each `SourceScopeActor`:

```
DecisionEvaluatorActor (rsi_oversold)
    │  decisionEvaluatedMessage (primitive data per DBI-9)
    ▼
MeanReversionEntryResolverActor
    │  1. Calls MeanReversionEntryResolver.Resolve() — pure function
    │  2. Validates strategy via Strategy.Validate()
    │  3. Constructs StrategyResolvedEvent with Metadata
    │  4. Sends publishStrategyMessage to StrategyPublisherActor
    ▼
StrategyPublisherActor
    │  Receives publishStrategyMessage
    │  Delegates to natsstrategy.Publisher.PublishStrategy()
    ▼
natsstrategy.Publisher
    │  1. Resolves EventSpec via specForType()
    │  2. Builds subject: strategy.events.mean_reversion_entry.resolved.{source}.{symbol}.{timeframe}
    │  3. Encodes event via natskit.EncodeEvent()
    │  4. Sets MsgID = Strategy.DeduplicationKey()
    │  5. Publishes to STRATEGY_EVENTS JetStream stream
    ▼
NATS JetStream (STRATEGY_EVENTS)
```

### 2.2 Resolution Logic (Pure Function)

`MeanReversionEntryResolver.Resolve()` is a pure function with no I/O:

| Decision Outcome | Strategy Direction | Confidence | Parameters |
|---|---|---|---|
| `"triggered"` | `long` | Scaled by severity (high=1.0×, moderate=0.90×, low=0.80×) | `entry=market`, `target_offset`, `stop_offset` adjusted by severity |
| `"not_triggered"` | `flat` | `"0.0000"` | None |
| `"insufficient"` | `flat` | `"0.0000"` | None, metadata carries `reason=insufficient_data` |
| Any other value | — | — | Returns `false` (no strategy produced) |

### 2.3 Event Construction

In `MeanReversionEntryResolverActor.onDecisionEvaluated()`:

```go
meta := events.NewMetadata().
    WithCorrelationID(msg.CorrelationID).
    WithCausationID(msg.CausationID)
event := domainstrategy.StrategyResolvedEvent{
    Metadata: meta,
    Strategy: strat,
}
```

Key properties:
- `Metadata.ID`: fresh UUID generated by `events.NewMetadata()`
- `Metadata.OccurredAt`: `time.Now().UTC()` (event creation time, not strategy timestamp)
- `Metadata.CorrelationID`: propagated from the decision's correlation chain
- `Metadata.CausationID`: set to the decision event's causation ID from fan-out

### 2.4 NATS Publication

`natsstrategy.Publisher.PublishStrategy()` produces:

| Property | Value |
|---|---|
| Subject | `strategy.events.mean_reversion_entry.resolved.{source}.{symbol}.{timeframe}` |
| Stream | `STRATEGY_EVENTS` |
| MsgID (dedup) | `strat:mean_reversion_entry:{source}:{symbol}:{timeframe}:{timestamp_unix}` |
| Envelope type | `strategy.events.v1.mean_reversion_entry_resolved` |
| CorrelationID | Passed through `natskit.EncodeEvent()` |
| CausationID | Passed through `natskit.EncodeEvent()` |

### 3.1 StrategyResolvedEvent Fields vs S359 Contract

| `metadata.correlation_id` | Propagated from `decisionEvaluatedMessage.CorrelationID` | **COMPLIANT** | End-to-end chain preserved |
| `metadata.causation_id` | Propagated from `decisionEvaluatedMessage.CausationID` | **COMPLIANT** | Points to decision event |
| `metadata.timestamp` (`occurred_at`) | `time.Now().UTC()` via `NewMetadata()` | **COMPLIANT** | Event creation time |
| `strategy.type` | `"mean_reversion_entry"` (hardcoded in resolver) | **COMPLIANT** | Matches S359 contract |

| `strategy.timeframe` | From `MeanReversionEntryResolver.timeframe` (binding config) | **COMPLIANT** | e.g., `60` |
| `strategy.direction` | From `Resolve()` logic: triggered→long, else→flat | **COMPLIANT** | Enum: long/short/flat |
| `strategy.confidence` | Severity-scaled from decision confidence | **COMPLIANT** | Decimal string [0.0, 1.0] |
| `strategy.decisions[]` | Single `DecisionInput` with type, outcome, confidence, severity, rationale, timeframe | **COMPLIANT** | Provenance preserved |

| `strategy.metadata` | `decision_type`, `decision_severity`, `rationale`, `decision_rationale` | **COMPLIANT** | Enriched context |
| `strategy.final` | Always `true` | **COMPLIANT** | All resolutions are final |
| `strategy.timestamp` | Propagated from `decisionEvaluatedMessage.Timestamp` | **COMPLIANT** | Decision-sourced timestamp |

### 3.2 S359 Invariant Compliance (INV-1 through INV-11)

| Invariant | Description | Producer Compliance | Evidence |
|---|---|---|---|
| **INV-1** | Type identity preservation | **COMPLIANT** | `strategy.Type = "mean_reversion_entry"` set by resolver; never altered in publisher chain |
| **INV-2** | Direction-to-side determinism | **N/A (consumer-side)** | Producer sets `Direction` deterministically; consumer maps to `Side`. Producer does not determine side. |
| **INV-3** | Causation chain integrity | **COMPLIANT** | `Metadata.CorrelationID` propagated from decision chain; `Metadata.CausationID` set from decision event ID. Consumer will set `ExecutionIntent.CausationID = event.Metadata.ID` (the strategy event's own ID). |
| **INV-4** | Pass-through risk is explicit | **N/A (consumer-side)** | Risk pass-through is set by `StrategyConsumerActor` in execute scope, not by derive producer. |
| **INV-5** | Timestamp monotonicity | **COMPLIANT** | `strategy.Timestamp` is set from `decisionEvaluatedMessage.Timestamp` (source-derived). Publisher does not substitute `time.Now()`. |
| **INV-6** | Single strategy family per consumer | **N/A (consumer-side)** | The producer publishes to the correct family-specific subject. Consumer subscription is consumer-side. |
| **INV-7** | Flat direction produces no execution | **COMPLIANT** | Resolver produces `Direction=flat` with `Confidence="0.0000"` for not_triggered/insufficient. Event is still published. Consumer handles no-execution semantics. |
| **INV-8** | Event acknowledgment after publication | **PARTIAL (producer-side)** | Publisher uses synchronous `js.Publish()` — returns only after JetStream confirms. If publish fails, error is logged and health tracker records error. Actor message delivery (not NATS ack) is the mechanism here. |
| **INV-9** | Kill switch precedence | **N/A (consumer-side)** | Kill switch is in `VenueAdapterActor`, downstream of execute. |
| **INV-10** | Staleness guard applies | **N/A (consumer-side)** | Staleness guard is in `VenueAdapterActor`, downstream of execute. |
| **INV-11** | Deduplication key uniqueness | **COMPLIANT** | `strat:{type}:{source}:{symbol}:{timeframe}:{timestamp_unix}` — unique per partition per second. Matches JetStream MsgID dedup. |

## 4. Ownership Model

### 4.1 Who Owns What

| Artifact | Owner | Binary | Invariant |
|---|---|---|---|
| `MeanReversionEntryResolver` (pure logic) | application/strategy | derive | SBI-4: no I/O, no actor refs |
| `MeanReversionEntryResolverActor` (actor wrapper) | actors/scopes/derive | derive | OR-6: lifecycle bound to binding |
| `StrategyPublisherActor` (NATS bridge) | actors/scopes/derive | derive | OR-7: one per scope |
| `natsstrategy.Publisher` (JetStream adapter) | adapters/nats/natsstrategy | derive | OR-1: single writer |
| `natsstrategy.Registry` (contract definitions) | adapters/nats/natsstrategy | shared | Defines subjects, streams, consumer specs |
| `Strategy` domain type | domain/strategy | shared | SBI-1: no upstream imports |
| `StrategyResolvedEvent` | domain/strategy | shared | Event envelope |
| `STRATEGY_EVENTS` JetStream stream | infra (created by publisher) | derive | Single-writer: only derive publishes |

### 4.2 Ownership Rules (from S53 strategy-activation-and-ownership)

| Rule | Description | Status |
|---|---|---|
| **OR-1** | Only derive writes to STRATEGY_EVENTS | **ENFORCED** — StrategyPublisherActor is the sole publisher |
| **OR-2** | Store reads STRATEGY_EVENTS per family | **ENFORCED** — StoreMeanReversionEntryStrategyConsumer durable |
| **OR-3** | Only store writes to strategy KV buckets | **ENFORCED** — StrategyProjectionActor in store |
| **OR-6** | StrategyResolverActor lifecycle bound to binding | **ENFORCED** — child of SourceScopeActor |
| **OR-7** | StrategyPublisherActor one per scope | **ENFORCED** — spawned per SourceScopeActor |

### 4.3 Domain Boundary Invariants (from S53 strategy-domain-design)

| Invariant | Status |
|---|---|
| **SBI-1**: strategy domain does not import decision/signal/evidence/observation | **ENFORCED** — `internal/domain/strategy/` has zero upstream domain imports |
| **SBI-2**: strategy receives decision data only as actor messages | **ENFORCED** — `decisionEvaluatedMessage` carries primitives, not `decision.Decision` |
| **SBI-3**: strategy publishes exclusively to STRATEGY_EVENTS | **ENFORCED** — `natsstrategy.Publisher` targets only STRATEGY_EVENTS |
| **SBI-4**: strategy resolver logic is pure | **ENFORCED** — `MeanReversionEntryResolver` has no I/O, no actor refs, no NATS |
| **SBI-9**: strategy consumes typed decision data, not raw Decision structs | **ENFORCED** — `DecisionInput` is strategy-owned type |
| **SBI-10**: strategy never consumes signals directly | **ENFORCED** — only `decisionEvaluatedMessage` input |

## 5. NATS Subject and Consumer Alignment

### 5.1 Subject Format

| Component | Value |
|---|---|
| Publisher subject pattern | `strategy.events.mean_reversion_entry.resolved.{source}.{symbol}.{timeframe}` |
| Execute consumer filter | `strategy.events.mean_reversion_entry.resolved.>` |
| Store consumer filter | `strategy.events.mean_reversion_entry.resolved.>` |
| Stream | `STRATEGY_EVENTS` (subjects: `strategy.events.>`) |

**Alignment**: The publisher's subject pattern matches both the execute and store consumer filters exactly. The `>` wildcard captures the `{source}.{symbol}.{timeframe}` suffix.

### 5.2 Consumer Spec Alignment

| Property | Execute Consumer (S360) | Registry Definition | Match |
|---|---|---|---|
| Durable | `execute-strategy-mean-reversion-entry` | `ExecuteStrategyMeanReversionEntryConsumer()` | YES |
| Subject | `strategy.events.mean_reversion_entry.resolved.>` | Same | YES |
| Stream | `STRATEGY_EVENTS` | Same | YES |
| AckWait | 30s | 30s | YES |
| MaxDeliver | 5 | 5 | YES |

### 5.3 Deduplication Key Analysis

| Domain | Key Format | Purpose |
|---|---|---|
| Producer (derive) | `strat:{type}:{source}:{symbol}:{timeframe}:{ts_unix}` | JetStream MsgID dedup — prevents duplicate strategy events in STRATEGY_EVENTS |
| Consumer (execute) | `exec:{type}:{source}:{symbol}:{timeframe}:{ts_unix}` | JetStream MsgID dedup — prevents duplicate execution intents in EXECUTION_EVENTS |

The prefixes differ by design: each domain generates its own dedup key for its own stream. This is correct and not a mismatch.

## 6. Correlation and Causation Chain

### 6.1 Full Chain Through Derive

```
observation → evidence → signal → decision → strategy
                                    │            │
                                    │            ├─ Metadata.CorrelationID: propagated from decision chain
                                    │            └─ Metadata.CausationID: set from decisionEvaluatedMessage.CausationID
                                    │
                                    ├─ CorrelationID: propagated from signal chain
                                    └─ CausationID: signal event ID
```

### 6.2 Correlation Propagation Path

| Hop | CorrelationID Source | CausationID Source |
|---|---|---|
| Signal → Decision fan-out | `signalGeneratedMessage.CorrelationID` | `signalGeneratedMessage.CausationID` (signal event ID) |
| Decision → Strategy fan-out | `decisionEvaluatedMessage.CorrelationID` | `decisionEvaluatedMessage.CausationID` (decision event ID) |
| Strategy event metadata | `events.NewMetadata().WithCorrelationID(msg.CorrelationID)` | `.WithCausationID(msg.CausationID)` |
| Strategy → Execute consumer | `event.Metadata.CorrelationID` | `event.Metadata.ID` (strategy event ID becomes execution's causation) |

The correlation chain originates at signal generation and propagates without alteration through every hop. Each hop sets its own `CausationID` to the upstream event's ID, creating an ordered causation chain.

## 7. Validation Gates

### 7.1 Producer-Side Validation

| Gate | Location | Behavior on Failure |
|---|---|---|
| `Resolve()` returns `false` | `MeanReversionEntryResolverActor.onDecisionEvaluated()` | Silent return — no event produced |
| `Strategy.Validate()` fails | `MeanReversionEntryResolverActor.onDecisionEvaluated()` | Error logged, no event produced |
| `natsstrategy.Publisher` unavailable | `StrategyPublisherActor.Receive()` | Health tracker error, log error |
| Unknown strategy type | `Publisher.specForType()` | Returns problem (InvalidArgument) |
| JetStream publish fails | `Publisher.PublishStrategy()` | Returns problem (Unavailable) |

### 7.2 Consumer-Side Validation (S360, for reference)

| Gate | Location | Behavior on Failure |
|---|---|---|
| Deserialization failure | `StrategyConsumerActor` | Message terminated (ack without processing) |
| `Strategy.Validate()` fails | `StrategyConsumerActor` | Message terminated |
| Wrong strategy type | `StrategyConsumerActor` | Message terminated |
| `PaperOrderEvaluator.Evaluate()` returns false | `StrategyConsumerActor` | Message ack'd (no error) |
| Execution publication fails | `StrategyConsumerActor` | Message NAK'd for redelivery |
