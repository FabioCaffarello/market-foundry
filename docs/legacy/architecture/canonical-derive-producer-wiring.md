# Canonical Derive Producer Wiring

> S366 deliverable. Documents how `StrategyResolvedEvent` is produced by the derive binary, the exact wiring path, integration points, and contracts preserved.

## Wiring Overview

The derive binary produces `StrategyResolvedEvent` through a four-stage actor chain within a single `SourceScopeActor` instance. No code changes were required in S366 — the wiring was already correct per the S365 compliance audit. This document captures the canonical wiring as validated.

## Producer Path

```
DecisionEvaluatorActor (e.g., RSIOversoldEvaluatorActor)
  │
  ├─ produces: publishDecisionMessage → DecisionPublisherActor (NATS)
  │
  └─ fan-out: decisionEvaluatedMessage → SourceScopeActor
       │
       └─ routes to: MeanReversionEntryResolverActor
            │
            ├─ calls: resolver.Resolve() — pure function, no I/O
            │
            ├─ validates: strat.Validate() — gates publication
            │
            ├─ constructs: StrategyResolvedEvent with:
            │    - Metadata.ID: fresh UUID
            │    - Metadata.CorrelationID: propagated from decision chain
            │    - Metadata.CausationID: propagated from decision fan-out
            │
            ├─ sends: publishStrategyMessage → StrategyPublisherActor
            │    │
            │    └─ natsstrategy.Publisher.PublishStrategy()
            │         │
            │         ├─ resolves EventSpec via specForType()
            │         ├─ constructs subject: {spec.Subject}.{source}.{symbol}.{timeframe}
            │         ├─ encodes: natskit.EncodeEvent (CBOR envelope)
            │         ├─ dedup key: Strategy.DeduplicationKey()
            │         └─ publishes: JetStream with MsgID
            │
            └─ fan-out: strategyResolvedMessage → SourceScopeActor → RiskEvaluatorActors
```

## Integration Points

### 1. Decision-to-Strategy Boundary (DBI-9)

| Property | Value |
|---|---|
| Transport | Actor message (`decisionEvaluatedMessage`) |
| Data format | Primitive fields only — no `decision.Decision` struct |
| Fields carried | Symbol, DecisionType, DecisionOutcome, DecisionConfidence, DecisionSeverity, DecisionRationale, Timeframe, Timestamp, CorrelationID, CausationID |

### 2. Strategy-to-Publisher Boundary

| Property | Value |
|---|---|
| Transport | Actor message (`publishStrategyMessage`) |
| Data format | Full `strategy.StrategyResolvedEvent` (same binary, same scope) |
| Validation | Already performed by resolver before sending |

### 3. Publisher-to-NATS Boundary

| Property | Value |
|---|---|
| Subject | `strategy.events.mean_reversion_entry.resolved.{source}.{symbol}.{timeframe}` |
| Stream | `STRATEGY_EVENTS` (FileStorage, 72h retention, 256 MB) |
| Encoding | CBOR via `natskit.EncodeEvent` |
| Dedup key | `strat:mean_reversion_entry:{source}:{symbol}:{timeframe}:{unix_ts}` |
| Timeout | 5 seconds per publish |

### 4. Strategy-to-Risk Fan-out Boundary (DBI-9)

| Property | Value |
|---|---|
| Transport | Actor message (`strategyResolvedMessage`) |
| Data format | Primitive fields only — no `strategy.Strategy` struct |
| Fields carried | Symbol, StrategyType, StrategyDirection, StrategyConfidence, DecisionSeverity, DecisionRationale, Timeframe, Timestamp, CorrelationID, CausationID |
| CausationID | Set to strategy event's `Metadata.ID` (not the decision's) |

## Correlation and Causation Flow

```
Signal Event (ID: sig-001, CorrelationID: trace-X)
  └─ Decision Event (ID: dec-001, CorrelationID: trace-X, CausationID: sig-001)
       └─ Strategy Event (ID: strat-001, CorrelationID: trace-X, CausationID: dec-001)
            └─ Risk Event (ID: risk-001, CorrelationID: trace-X, CausationID: strat-001)
                 └─ Execution Event (ID: exec-001, CorrelationID: trace-X, CausationID: risk-001)
```

- **CorrelationID**: Immutable — propagated from the original observation, never generated by strategy.
- **CausationID**: DAG linkage — each event's CausationID equals its parent event's Metadata.ID.

## Consumer Alignment

The producer's subject pattern aligns with all registered consumers:

| Consumer | Durable | Subject Filter |
|---|---|---|
| Writer (store) | `writer-strategy-mean-reversion-entry` | `strategy.events.mean_reversion_entry.resolved.>` |
| Store | `store-strategy-mean-reversion-entry` | `strategy.events.mean_reversion_entry.resolved.>` |
| Execute | `execute-strategy-mean-reversion-entry` | `strategy.events.mean_reversion_entry.resolved.>` |

All consumers use `>` wildcard, which matches the producer's `{source}.{symbol}.{timeframe}` suffix.

## Activation Path

Strategy resolution activates when all conditions are met:

1. `pipeline.strategy_families` includes `"mean_reversion_entry"` in derive config
2. `BindingWatcherActor` detects active binding (startup query or runtime event)
3. `DeriveSupervisor.ensureSourceScope()` creates `SourceScopeActor` for the source
4. `SourceScopeActor` spawns `MeanReversionEntryResolverActor` + `StrategyPublisherActor`
5. Upstream signal and decision actors produce events through the pipeline
