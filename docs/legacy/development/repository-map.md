# Repository Map

## Purpose

This file routes contributors to the right physical area of the repository
without making them scan the tree blindly.

## Start By Task

| Task | Start here |
|---|---|
| Understand binaries and service entrypoints | [`../../cmd/README.md`](../../cmd/README.md) |
| Understand implementation layers and placement | [`../../internal/README.md`](../../internal/README.md) |
| Inspect compose, configs, envs, or migrations | [`../../deploy/README.md`](../../deploy/README.md) |
| Inspect scripts behind `make` | [`../../scripts/README.md`](../../scripts/README.md) |
| Inspect repository-level test assets | [`../../tests/README.md`](../../tests/README.md) |
| Inspect CLI/tooling internals | [`../tooling/README.md`](../tooling/README.md) |
| Inspect product and architecture context first | [`../product/README.md`](../product/README.md) |

## Documentation Roles

| Surface | Role |
|---|---|
| `docs/product/` | Product and system context for humans |
| `docs/development/` | Canonical contributor workflow and navigation |
| `docs/tooling/` | Tool-internal reference and guardrails |
| `docs/architecture/` | Deep canonical architecture corpus |
| `docs/stages/` | Historical evidence trail |
| `docs/archive/` | Legacy and superseded material |
