# Security Policy

## Reporting a Vulnerability

This is a personal project maintained by a single developer. While
external contributions (PRs, forks) are not accepted, security reports
are welcome.

### How to report

If you believe you've found a security vulnerability in this code:

- **Do not** open a public issue.
- **Do** send a description to the maintainer via the email address
  associated with the GitHub profile (see commit history for the email).

Include:
- A description of the vulnerability.
- Steps to reproduce, if applicable.
- Affected versions / commits.
- Any suggested mitigation.

### What to expect

Acknowledgment within reasonable time (best effort — this is a personal
project, no SLA). If the report is valid and applicable, a fix will be
prepared. No bug bounty program.

### Scope

This policy covers code in this repository. It does NOT cover:

- Dependencies (report upstream).
- Forks or copies elsewhere.
- Misconfiguration in user-side deployments.
